## 2024-04-03 - Accessible File Inputs and Modals
**Learning:** Custom file inputs that use `display: none` or Tailwind's `hidden` are completely removed from the accessibility tree and keyboard tab order, breaking usability for screen readers and keyboard users. Additionally, custom modal implementations often lack critical accessibility roles (`dialog`, `aria-modal`) and explicit focus indicators for interaction elements.
**Action:** Always use the `sr-only peer` pattern on `<input type="file">` instead of `hidden`, and style the associated `<label>` with `peer-focus-visible` classes to maintain focus visibility while hiding the ugly default file input visually. Ensure all custom modals receive `role="dialog"`, `aria-modal="true"`, and proper `aria-labelledby` IDs, along with clear focus rings on all interactive buttons (including icon-only close buttons).
