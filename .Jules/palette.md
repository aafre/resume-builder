## 2025-02-14 - ResponsiveConfirmDialog for Destructive Actions
**Learning:** Destructive actions (like Delete) implemented with custom hardcoded modals lack standard accessibility attributes (`role="dialog"`, `aria-modal`, etc.) and mobile responsiveness (like bottom sheets). This app has a `ResponsiveConfirmDialog` component designed specifically for this purpose, but it was not being utilized uniformly.
**Action:** Always use `ResponsiveConfirmDialog` for destructive confirmation prompts (such as `DeleteResumeModal`) to ensure a consistent, accessible, and mobile-friendly UX that prevents accidental data loss.
## 2025-03-05 - Invisible Focus States on Drag-and-Drop List Controls
**Learning:** Interactive controls (like delete buttons or add item buttons) rendered inside dynamically mapped lists (e.g., `ExperienceItem`, `GenericSection`) frequently lose default browser focus outlines due to custom Tailwind padding, rounded corners, or background colors. This creates a critical accessibility issue where keyboard users cannot see which item they are about to delete.
**Action:** Always explicitly add `focus-visible:outline-none focus-visible:ring-2` to list manipulation controls. Use `focus-visible:ring-red-500` for destructive actions and `focus-visible:ring-accent` for additive actions, accompanied by `focus-visible:ring-offset-1` or `2` to ensure contrast against list item backgrounds.
