## 2024-03-24 - Parallelizing FileReader Conversions in Icon Exports
**Learning:** Using a sequential `for...of` loop with `await` for multiple independent asynchronous operations, specifically `FileReader` conversions during icon exports in `exportIconsForYAML`, is a performance anti-pattern. This approach blocks the loop, causing the total execution time to scale linearly with the number of icons, unnecessarily delaying the export process when these I/O operations could be executed concurrently.
**Action:** When handling multiple independent file conversions (like reading files to base64 strings), refactor sequential `await` loops into an array of promises mapped from the items, and await them concurrently using `Promise.all()`. Maintain individual `try/catch` blocks within the mapping function to ensure a single failure doesn't abort the entire batch operation.
