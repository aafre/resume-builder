## 2025-01-20 - Concurrent Async Initialization using Promise.all in useIconRegistry
**Learning:** Sequential `for...of` loops using `await` inside UI hooks (such as when parsing or serializing multiple user-uploaded assets) cause unnecessary waterfall delays, particularly for I/O-bound operations like `FileReader.readAsDataURL()`. We found this pattern in `useIconRegistry.ts` during `exportIconsForYAML`.
**Action:** Refactor sequential asynchronous sets into concurrent operations by wrapping a `.map` array inside `Promise.all()`. This pattern drastically reduces total execution time. Crucially, always wrap the inner function in a `try/catch` block to ensure that a single failing parsing operation does not reject the entire `Promise.all` operation and break the user's export.
